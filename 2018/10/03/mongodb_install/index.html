MongoDB的安装与配置

发表于 2018-10-03 | 分类于 mongodb

1.MongoDB的安装与配置

下载地址：https://www.mongodb.com/download-center#community
下载完安装包，并解压 tgz（以下演示的是 64 位 Linux上的安装） 。
下面以 ubuntu为例

  • 解压 
    tar -zxvf mongodb-linux-x86_64-4.0.2.tgz
  • 移动到/usr/local/目录下
    sudo mv -r mongodb-linux-x86_64-ubuntu1604-4.0.2/ /usr/local/mongodb
  • 将可执行文件添加到PATH路径中
    export PATH=/usr/local/mongodb/bin:$PATH
    创建数据库目录
    MongoDB的数据存储在data目录的db目录下，但是这个目录在安装过程不会自动创建，所以你需要手动创建data目录，并在data目录中创建db目录。
    注意：/data/db 是 MongoDB 默认的启动的数据库路径(–dbpath)。
sudo mkdir /data/db
cd /data
sudo chmod -R 777 db

2.管理mongo

  • 默认端口27017
  • 启动
    进入mongodb的bin目录（我的是/usr/local/mongodb/bin）
    ./mongod
  • 关闭
    ./mongod --shutdown

另外还可以使用配置文件启动mongodb

  • 创建配置文件
    sudo mkdir /etc/mongodb/mongod.conf

  • 创建日志文件

    sudo mkdir /etc/mongodb/mongod.log

    以上的路径按照自己喜欢创建

  • 编辑配置文件

# 配置日志文件路径
logpath = /etc/mongodb/mongod.log
# 日志追加
logappend = true
# 数据库目录
dbpath = /data/db
# 以守护进程启动
fork = true

  • 配置文件启动方式

    ./mongod -f /etc/mongodb/mongod.conf

  • 查看进程

    ps -el | grep mongodb

  • MongoDB后台管理 Shell
    如果你需要进入MongoDB后台管理，你需要先打开mongodb装目录的下的bin目录，然后执行mongo命令文件。

MongoDB Shell是MongoDB自带的交互式Javascript shell,用来对MongoDB进行操作和管理的交互式环境。

cd /usr/local/mongodb/bin
./mongo
# mongodb
